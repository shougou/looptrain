# Idea: 新手阶段渐进解锁 UI 优化

**提出日期**: 2026-06-25
**关联版本**: v0.11.0-mobile-portrait-ui-redesign
**来源**: 综合 `my_idea.md` 设计提案 + `ui-ux-audit-report-v0.11.0.md` 审查结论

---

## 1. 问题陈述

v0.11.0 完成了重要的工程重构：组件化、四区布局、ActionResultCard、底部行动区、档案抽屉、对话聚焦模式均已具备。但通过实际浏览器审查发现，当前首屏同时暴露 15+ 个控件元素，导致玩家第一眼无法判断"我现在应该干什么"。

核心矛盾：**UI 已经是一个完整的系统控制台，但新手第一分钟只需要知道三件事**：我在哪？发生了什么异常？我下一步点哪个按钮？

---

## 2. 核心目标

把 LoopTrain 从"复杂系统控制台"改成"许知微引导的循环推理剧场"。

具体而言：**默认只展示玩家当前必须知道的信息；复杂系统通过许知微、案件板、更多行动逐步展开。**

---

## 3. 关键设计原则

| 原则 | 说明 |
|------|------|
| **渐进暴露** | 第一轮只给 1 个行动按钮，第二轮 2 个，第三轮后才开放完整系统 |
| **消除工程语言** | 删除"新版试玩已重构""AP""高风险行动"等开发者术语 |
| **上下文按钮** | 所有按钮必须回答"点了以后会发生什么"，不使用泛化确认文案 |
| **许知微主界面化** | 从顶部工具按钮改为剧情提示卡，承担新手导航 + 推理副驾驶角色 |
| **案件板替代档案** | 默认打开不是线索/人物/时间线 Tab，而是"本轮确认/NPC说法/待验证/关键矛盾" |

---

## 4. 与审查报告的对齐

本次 work item 覆盖了审查报告中 **所有 12 个问题**，并提升了修复策略：

| 审查问题 | 本方案策略 |
|---------|-----------|
| P0-1 开场覆盖层遮挡 | 首屏只展示状态+场景+许知微提示+1个按钮，消除层级混淆 |
| P0-2 设置面板单薄 | 设置保留为最小齿轮图标，不争夺首屏注意力 |
| P1-1 Emoji 不统一 | 渐进解锁阶段中，顶部只保留案件板入口（无 Emoji） |
| P1-2 输入框体验差 | 默认隐藏输入框，只有自定义问题时才显示 |
| P1-3 档案标签无反馈 | 档案改为案件板，默认展示结构化推理卡片而非 Tab |
| P1-4 更多行动层级不清 | 第一轮不出现"更多行动"，后续按阶段解锁 |
| P1-5 NPC 状态无颜色 | 许知微提示卡中自然融入 NPC 状态描述，无需额外标签 |
| P2-1 图标风格不一致 | 首屏极简后，顶部只剩状态栏+设置齿轮，风格统一 |
| P2-2 缺少加载状态 | 行动按钮点击后增加局部加载态（按钮变灰+旋转） |
| P2-3 时间轴细节不足 | 第一轮改为"爆炸倒计时"压力条，第二轮显示上一轮记忆 |
| P2-4 目标面板遮挡 | 目标改为"当前目标一句话"，不展示进度清单 |
| P2-5 文本排版层次 | 场景描述精简为"位置+当前异常+当前目标"三段式 |

## 5. 具体任务

### 任务 1：清理首屏工程语言
- 删除游戏内"新版试玩已重构"等工程术语
- 入口改为：LoopTrain：寒灯初醒 / [开始循环] / [继续上次循环]

### 任务 2：隐藏第一轮非必要控件
第一轮隐藏：档案、更多行动、高风险行动、对话/行动 Tab、发送、完整时间线、重置游戏

### 任务 3：引入 UIStage 状态机
前端增加 `UIStage` 字段（intro → first_observation → first_dialogue → loop_memory_intro → caseboard_intro → contradiction_intro → normal_play），不同阶段决定可见控件

### 任务 4：重写 ActionDock 规则
- 第一轮 1 个主按钮
- 第二轮 2 个主按钮
- 第三轮以后 2-3 个主按钮 + 更多

### 任务 5：许知微主界面化
从顶部工具按钮改为剧情提示卡，根据游戏状态动态生成提示内容

### 任务 6：档案改为案件板
ArchiveSheet 默认页改成 CaseBoard（本轮确认 / NPC 说法 / 待验证 / 关键矛盾 / 下一步建议），底部保留 [线索] [人物] [时间线] [记忆] 二级入口

### 任务 7：高风险行动上下文化
隐藏全局高风险按钮，只有具体条件满足时显示具体高风险动作（如"强行搜查灰衣乘客行李"），并附带后果确认弹窗

### 任务 8：加载状态与交互反馈
- 按钮点击后局部加载态（变灰 + 旋转图标）
- 菜单展开/收起动画过渡
- 面板滑入/滑出动画

## 6. 信息架构（三层）

### 第一层：玩家主界面（默认）
```
当前时间 + 倒计时
当前现场（位置 / 异常 / 目标）
许知微提示
最近一次行动结果
1-3 个推荐行动
```

### 第二层：案件板（玩家主动进入）
```
本轮确认
NPC 说法
待验证
关键矛盾
下一步建议
```

### 第三层：完整推理系统（中后期开放）
```
线索列表 / 人物状态 / 完整时间线 / 上一轮记忆 / 证据评分
```

## 7. 验收标准

### 新手 30 秒验收
玩家第一次进入后，不需要任何说明，能回答：
- 我在列车上
- 时间很紧
- 我应该先观察当前车厢

### 新手 3 分钟验收
完成第一次观察和询问后，能回答：
- 小宁可能有问题
- 她说的话不一定是真的
- 我需要验证她有没有离开过座位

### 第一轮失败验收
玩家失败后能回答：
- 我不是白失败
- 下一轮我可以带着上一轮记忆提前盯住她

### 第二轮验收
玩家自然点击"盯住小宁"，而不是打开档案乱翻
