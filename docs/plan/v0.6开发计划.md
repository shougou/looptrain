# LoopTrain v0.6 开发计划

**版本**: v0.6 结构化 Runtime  
**基准**: v0.5 可玩 standalone 实现已经完成  
**目标**: 按照架构文档迁移到分层 TypeScript 架构

---

## 背景

v0.5 已经完成：
- 纯 JavaScript 可玩
- 消息音效 + UX 优化完成
- 满足核心规则：Engine 判断事实，LLM 只做表达，Assistant 只基于 CompanionView 输出

---

## 开发路线

### Phase 0: 项目初始化 🔴

**必须完成** 才能进入下一阶段：

- [ ] 创建目录结构
  ```
  mkdir -p looptrain/standalone/src/runtime/{shared,ids,engine,memory,companion-view,assistant,content,policy,tests}
  ```

- [ ] 创建 `tsconfig.runtime.json` - 参考 spec 文档配置
- [ ] 更新 `package.json` 添加 `build:runtime` / `test:runtime` / `test:standalone` 脚本
- [ ] 验证 `node --check` 通过

### Phase 1: 核心接口 🔶

- [ ] 定义 shared 模块
  - `ids.ts` - Runtime ID 生成 / 校验
  - `time.ts` - 时间工具
  - `errors.ts` - 错误类型
  - `result.ts` - EngineResult 类型

- [ ] 定义 engine 模块
  - `EngineResult.ts` - 引擎结果类型
  - `MemoryEventDraft.ts` - 内存事件草稿类型
  - `LegacyEngineAdapter.ts` - 旧引擎桥接接口（必须实现 spec 契约）

- [ ] 定义 memory 模块
  - `MemoryEvent.ts` - 内存事件类型
  - `RuntimeClientState.ts` - 客户端状态契约
  - `SerializedMemoryState.ts` - 序列化状态

- [ ] 定义 companion-view 模块
  - `CompanionView.ts` - 视图接口
  - `CompanionBuilder.ts` - 构建器
  - `CompanionPolicy.ts` - 可见性策略
  - `CompanionVisibilityFilter.ts` - 可见性过滤
  - `CompanionSpoilerGuard.ts` - 防剧透守卫

- [ ] 定义 assistant 模块
  - `index.ts` - 主控制器
  - `AssistantTypes.ts` - 类型定义
  - `IntentClassifier.ts` - 意图分类
  - `AssistantPolicyEngine.ts` - 策略引擎
  - `ActionRegistry.ts` - 动作注册表
  - `ActionPlanner.ts` - 动作规划器
  - `AssistantSettlementReader.ts` - 结算读取
  - `PromptBuilder.ts` - Prompt 构造
  - `LLMProvider.ts` - LLM 提供者接口
  - `MockLLMProvider.ts` - Mock 实现
  - `DisabledLLMProvider.ts` - Disabled 实现
  - `DeepSeekLLMProvider.ts` - DeepSeek 实现
  - `OutputValidator.ts` - 输出验证
  - `ResponseRenderer.ts` - 结果渲染
  - `FallbackTemplateEngine.ts` - 降级模板

- [ ] 定义 content 模块
  - `RuntimeContentLoader.ts` - 运行时内容加载器
  - `ContentPathPolicy.ts` - 内容路径策略

- [ ] 定义 policy 模块
  - `SpoilerPolicy.ts` - 防剧透策略
  - `ForbiddenRevealPolicy.ts` - 禁止揭示策略

### Phase 2: 组装入口 🟡

- [ ] 编写 `looptrain/standalone/src/runtime/index.ts` 导出公共接口
- [ ] 更新 `server.js` 引入编译后的模块
- [ ] 确保 `looptrain/standalone/dist/runtime/index.js` 编译输出（git 忽略）
- [ ] 测试 `node --check` 通过

### Phase 3: 测试 ✅

- [ ] 创建 `looptrain/standalone/src/runtime/tests/` 测试骨架
- [ ] 编写基础模块单元测试
  - 验证 `first_contact_empty_memory` 能跑通
  - 验证 `loop1_after_failed_questioning` 能跑通
- [ ] 验证 `npm run build:runtime` 编译成功
- [ ] 验证 `npm run test:runtime && npm run test:standalone` 全部测试通过

---

## 验收标准

**完成** 必须满足：

1. ✅ 编译 `npm run build:runtime` 成功
2. ✅ 测试 `npm run test:runtime && npm run test:standalone` 全部通过
3. ✅ `node --check looptrain/standalone/dist/runtime/index.js` 语法正确
4. ✅ `server.js` 能正常 require 新模块
5. ✅ 所有原有 API 保持兼容，dev 可以正常启动游戏
