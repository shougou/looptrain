下面给一版**面向 0.7 试玩版**的重构设计。重点不是做完整剧情，而是让玩家在 15–25 分钟内明确理解 LT 的核心玩法：

> **行动目标引导 → 自由指令行动 → 获得结构化线索 → 记忆带入下一轮 → 许知微复盘 → 下一轮路线优化。**

---

# 一、试玩版定位

试玩版不要追求“大悬疑完整闭环”，而应该是一个**小而硬的系统样本**。

核心体验目标只有 5 个：

1. 玩家知道自己在循环中。
2. 玩家知道可以通过自然语言或快捷行动执行指令。
3. 玩家知道线索会进入记忆，并带入下一轮。
4. 玩家知道目标完成不是靠对话文本，而是靠结构化线索和事件。
5. 玩家知道许知微不是系统提示，而是剧情内助手。

试玩版建议控制在：

```text
循环次数：3 轮
游戏内时间：每轮 15 分钟
NPC 数量：3 个主要 NPC + 1 个可疑人物
核心线索：8 条以内
行动目标：6~8 个
最终试玩目标：提前定位危险物，并完成一次有效干预
```

---

# 二、故事背景重构

## 故事名称

可以暂定为：

> **《LoopTrain：寒灯初醒》**

这是正式版第一章的试玩切片。

---

## 背景设定

时间：**1939 年冬，江城号列车。**

主角在一节夜行列车上醒来，头痛、失忆，只记得一个非常明确的信息：

> **15 分钟后，这列车会爆炸。**

他不知道自己是谁，也不知道为什么知道这件事。

身边有一个年轻女性，名叫**许知微**。她自称是报社记者，但她似乎比普通乘客更冷静，也更快接受“循环”这件事。

许知微告诉主角：

> “你每次醒来，都会回到同一刻。你会忘掉一部分细节，但我会帮你记下来。”

这个设定把助手系统自然嵌入剧情。

---

# 三、开场白设计

开场不要解释太多。玩家一开始只需要知道三件事：

```text
我在列车上。
15 分钟后会爆炸。
许知微能帮我记录和复盘。
```

## 开场字幕

```text
铁轨的震动从脚下传来。

你睁开眼，窗外是浓重的夜色。
车厢灯光昏黄，空气里有煤烟、皮革和潮湿木板的气味。

你的太阳穴剧烈跳动。
脑子里只剩下一句话：

十五分钟后，这列车会爆炸。

你不知道这句话从何而来。
也不知道自己为什么会在这里。

对面的年轻女人合上笔记本，盯着你看了几秒。

她低声说：

“你又醒了。”
```

---

## 许知微第一次主动对话

```text
许知微：
“先别急着证明自己是不是疯了。我们已经试过几次了。”

“你每次都会在这个时间醒来。十五分钟后，爆炸发生。然后一切重来。”

“你会丢失一些细节，但我记录下来的东西还在。”

“这一轮不要试图一次解决所有问题。先确认三件事：你在哪里，爆炸什么时候发生，身边谁见过异常。”
```

此时系统展示第一个行动目标：

```text
当前目标：
确认当前环境和爆炸前的基础信息

建议行动：
[观察车厢] [询问许知微] [询问小宁] [查看当前时间]
```

---

# 四、人物设计

试玩版人物不要多。建议只放 4 个。

---

## 1. 主角：寒灯

表层身份：失忆乘客。
真实身份：暂不揭示，只留下代号。

玩家暂时只知道：

```text
你不是普通乘客。
你知道爆炸会发生。
你似乎曾经执行过某个任务。
“寒灯”这个代号会在试玩版最后出现。
```

设计作用：

* 承载玩家视角；
* 不需要复杂背景；
* 通过循环逐步恢复身份。

---

## 2. 许知微

表层身份：年轻记者。
系统职责：助手 Runtime 的剧情化载体。

她承担四个功能：

```text
1. 新手引导
2. 行动目标解释
3. 线索整理
4. 每轮失败复盘
```

人物性格建议：

* 冷静；
* 观察力强；
* 不过度安慰；
* 说话像记者和调查员，不像系统客服。

示例语气：

```text
“这条信息有价值。它把异常范围缩小到了三号车厢。”

“我们不需要重复上一轮已经确认的事。下一轮可以直接去验证那个帆布包。”

“别急着抓人。现在只有行为异常，还没有证据。”
```

---

## 3. 小宁

身份：十几岁的女孩，和母亲一起乘车。
作用：第一条稳定线索来源。

她看到过异常，但一开始不愿意说完整。

她能提供：

```text
灰衣乘客
金属碰撞声
三号车厢行李架
旧帆布包
```

设计价值：

* 教玩家“询问 NPC”；
* 教玩家“同一 NPC 可以追问不同 topic”；
* 教玩家“线索可带入下一轮”。

---

## 4. 陆成

身份：列车乘务员。
作用：干预行动的关键 NPC。

他一开始不相信主角，但如果玩家已经获得足够线索，就可以说服他。

他能提供：

```text
三号车厢钥匙
乘客调动权限
临时清空车厢
列车紧急处置流程
```

设计价值：

* 教玩家不是所有目标都靠调查完成；
* 有些目标需要“证据组合”才能解锁行动。

---

## 5. 灰衣乘客

身份：可疑人物。
试玩版不揭示真实阵营。

玩家只需要知道：

```text
他在爆炸前进入三号车厢。
他放置过一个旧帆布包。
他不一定是最终幕后黑手。
```

设计价值：

* 作为第一章悬疑钩子；
* 引导玩家理解“可疑不等于真相”。

---

# 五、试玩版时间线

每轮 15 分钟，固定从 **14:00 到 14:15**。

## 世界时间线

| 时间    | 事件                 |
| ----- | ------------------ |
| 14:00 | 主角醒来，许知微提醒循环已开始    |
| 14:02 | 小宁从三号车厢方向回来，神情紧张   |
| 14:04 | 灰衣乘客经过二号车厢，手里没有包   |
| 14:06 | 灰衣乘客进入三号车厢         |
| 14:08 | 三号车厢行李架传出轻微金属碰撞声   |
| 14:10 | 陆成经过二号车厢，可被拦下      |
| 14:12 | 灰衣乘客回到二号车厢，手里仍然没有包 |
| 14:13 | 列车进入山洞，车厢灯光短暂闪烁    |
| 14:15 | 爆炸发生，循环重置          |

这个时间线要非常稳定。玩家可以通过不同路线发现其中一部分。

# 六、三轮引导节奏

## 第一轮：认识系统

目标：让玩家知道可以观察、询问、获得线索、失败后复盘。

### 第一轮主动引导

许知微：

```text
“这一轮不要想着阻止爆炸。先确认基础信息。”

“观察环境，问身边的人，弄清楚爆炸前有没有异常。”
```

### 第一轮目标

```text
目标 1：观察当前车厢
目标 2：询问小宁是否看到异常
目标 3：确认爆炸时间
```

### 第一轮预期收获

玩家大概率获得：

```text
clue.explosion_time_1415
clue.xiaoning_nervous
clue.xiaoning_heard_metal_sound
```

第一轮允许失败。失败后进入复盘。

### 第一轮结束复盘

```text
许知微：
“这一轮没有白费。”

“我们确认了三件事：爆炸固定发生在十四点十五分；小宁在爆炸前去过三号车厢；她听到过金属碰撞声。”

“下一轮不要从头问起。直接围绕三号车厢调查。”
```

---

## 第二轮：理解记忆带入

目标：让玩家感受到上一轮信息可以优化行动路线。

### 第二轮开场

```text
许知微：
“上一轮的记录还在。”

“我们已经知道小宁听到声音，方向在三号车厢。这次直接验证三号车厢，不要浪费时间。”
```

### 第二轮目标

```text
目标 4：前往三号车厢
目标 5：检查行李架
目标 6：确认旧帆布包
```

### 第二轮预期收获

```text
clue.carriage_3_suspicious
clue.canvas_bag_on_rack
clue.gray_passenger_entered_carriage_3
```

第二轮仍然可以失败，但玩家会明显推进。

### 第二轮结束复盘

```text
许知微：
“现在危险物的范围已经缩小。”

“三号车厢，行李架，旧帆布包。灰衣乘客在十四点零六分进入那里，十四点十二分回来，手里没有包。”

“下一轮的重点不是继续观察，而是干预。”
```

---

## 第三轮：完成一次有效干预

目标：让玩家理解“线索组合 → 解锁行动 → 改变结果”。

### 第三轮开场

```text
许知微：
“这次我们已经不需要证明爆炸存在。”

“要做的是拿着足够明确的线索，让陆成相信我们，并清空三号车厢。”
```

### 第三轮目标

```text
目标 7：用已知线索说服陆成
目标 8：清空三号车厢
目标 9：完成一次有效干预
```

### 试玩版成功条件

不是“拆弹”，而是：

```text
在 14:15 前，让陆成清空三号车厢，并隔离旧帆布包。
```

成功后爆炸可以仍然发生，但结果改变：

```text
爆炸声仍然响起。

但这一次，三号车厢是空的。

许知微扶着车厢门框，脸色发白，却没有倒下。

她看着你，低声说：

“寒灯。”

“你刚才的反应，不像普通乘客。”
```

试玩版结束。

这个结尾既给玩家正反馈，又留下正式版钩子。

# 七、指令系统设计

试玩版指令系统不要开放太多能力。先稳定支持 7 类动作。

## 标准 Action 类型

```ts
type ActionType =
  | "OBSERVE"
  | "ASK_NPC"
  | "MOVE_TO"
  | "INSPECT_OBJECT"
  | "FOLLOW_NPC"
  | "PERSUADE_NPC"
  | "ASK_ASSISTANT"
  | "REVIEW_MEMORY"
```

## 玩家输入归一示例

| 玩家自然语言     | 标准 Action                                  |
| ---------- | ------------------------------------------ |
| 看看车厢里有什么   | OBSERVE current_carriage                   |
| 问小宁刚才看到了什么 | ASK_NPC xiaoning abnormal_event            |
| 去三号车厢      | MOVE_TO carriage_3                         |
| 检查行李架      | INSPECT_OBJECT luggage_rack                |
| 跟着灰衣人      | FOLLOW_NPC gray_passenger                  |
| 让乘务员清空三号车厢 | PERSUADE_NPC conductor evacuate_carriage_3 |
| 问许知微现在该怎么办 | ASK_ASSISTANT current_goal                 |
| 查看上一轮线索    | REVIEW_MEMORY known_clues                  |

关键点：

> 指令系统可以用 LLM 辅助解析，但最后必须归一到标准 Action。
> Runtime 只执行标准 Action。
> GoalEngine 只读取标准 Event / Clue / Fact / State。

# 八、核心线索设计

试玩版控制在 8 条线索以内。

| 线索 ID                                    | 线索内容          | 获取方式         | 作用     |
| ---------------------------------------- | ------------- | ------------ | ------ |
| clue.explosion_time_1415                 | 爆炸固定发生在 14:15 | 第一轮失败或查看怀表   | 建立循环规则 |
| clue.xiaoning_nervous                    | 小宁从三号车厢回来后紧张  | 观察小宁         | 引导询问   |
| clue.xiaoning_heard_metal_sound          | 小宁听到金属碰撞声     | 询问小宁异常声音     | 指向三号车厢 |
| clue.gray_passenger_entered_carriage_3   | 灰衣乘客进入三号车厢    | 询问小宁 / 跟踪灰衣人 | 锁定可疑人物 |
| clue.canvas_bag_on_luggage_rack          | 三号车厢行李架有旧帆布包  | 检查三号车厢行李架    | 锁定危险物  |
| clue.gray_passenger_returned_without_bag | 灰衣人回来时手里没有包   | 观察 / 跟踪      | 形成证据链  |
| clue.conductor_can_clear_carriage        | 陆成有权清空车厢      | 询问陆成 / 许知微提示 | 解锁干预   |
| clue.enough_evidence_for_conductor       | 线索足以说服陆成      | 组合线索自动生成     | 解锁最终行动 |

最后一条是派生线索，不是直接发现。

```text
当玩家同时拥有：
- clue.canvas_bag_on_luggage_rack
- clue.gray_passenger_entered_carriage_3
- clue.gray_passenger_returned_without_bag

则自动生成：
clue.enough_evidence_for_conductor
```

这能体现 GoalEngine / MemoryRuntime 的价值。

# 九、行动目标设计

试玩版目标建议分为三类：

```text
教学目标
调查目标
干预目标
```

## 目标 1：确认当前环境

```ts
{
  id: "goal.observe_current_carriage",
  title: "确认当前环境",
  loopRange: [1, 1],
  completionCondition: {
    eventOccurred: {
      type: "OBSERVE",
      target: "current_carriage"
    }
  },
  feedback: "你确认自己位于二号车厢，距离爆炸还有十五分钟。"
}
```

作用：教玩家观察。

---

## 目标 2：询问小宁异常情况

```ts
{
  id: "goal.ask_xiaoning_abnormal",
  title: "询问小宁是否看到异常",
  loopRange: [1, 2],
  completionCondition: {
    any: [
      { clueKnown: "clue.xiaoning_heard_metal_sound" },
      { clueKnown: "clue.gray_passenger_entered_carriage_3" }
    ]
  },
  feedback: "小宁提供了爆炸前的异常信息。"
}
```

作用：教玩家对话目标不是“聊过”，而是获得有效线索。

---

## 目标 3：确认爆炸时间

```ts
{
  id: "goal.confirm_explosion_time",
  title: "确认爆炸发生时间",
  completionCondition: {
    clueKnown: "clue.explosion_time_1415"
  }
}
```

作用：建立循环规则。

---

## 目标 4：前往三号车厢

```ts
{
  id: "goal.move_to_carriage_3",
  title: "前往三号车厢",
  activationCondition: {
    clueKnown: "clue.xiaoning_heard_metal_sound"
  },
  completionCondition: {
    stateEquals: {
      key: "player.location",
      value: "carriage_3"
    }
  }
}
```

作用：让线索推动路线变化。

---

## 目标 5：检查三号车厢行李架

```ts
{
  id: "goal.inspect_luggage_rack",
  title: "检查三号车厢行李架",
  activationCondition: {
    stateEquals: {
      key: "player.location",
      value: "carriage_3"
    }
  },
  completionCondition: {
    clueKnown: "clue.canvas_bag_on_luggage_rack"
  }
}
```

作用：让玩家理解观察和检查不同。

---

## 目标 6：确认灰衣乘客与帆布包有关

```ts
{
  id: "goal.link_gray_passenger_to_bag",
  title: "确认灰衣乘客与旧帆布包有关",
  completionCondition: {
    all: [
      { clueKnown: "clue.gray_passenger_entered_carriage_3" },
      { clueKnown: "clue.gray_passenger_returned_without_bag" },
      { clueKnown: "clue.canvas_bag_on_luggage_rack" }
    ]
  }
}
```

作用：教玩家线索组合。

---

## 目标 7：说服陆成清空三号车厢

```ts
{
  id: "goal.persuade_conductor",
  title: "说服陆成清空三号车厢",
  activationCondition: {
    clueKnown: "clue.enough_evidence_for_conductor"
  },
  completionCondition: {
    eventOccurred: {
      type: "PERSUADE_NPC",
      npcId: "conductor_lucheng",
      topic: "evacuate_carriage_3",
      result: "success"
    }
  }
}
```

作用：教玩家有些行动需要前置证据。

---

## 目标 8：完成有效干预

```ts
{
  id: "goal.complete_demo_intervention",
  title: "完成一次有效干预",
  completionCondition: {
    all: [
      { stateEquals: { key: "carriage_3.evacuated", value: true } },
      { stateEquals: { key: "canvas_bag.isolated", value: true } },
      { stateBefore: { key: "world.time", value: "14:15" } }
    ]
  }
}
```

作用：试玩版通关目标。

# 十、MemoryRuntime 设计

试玩版 MemoryRuntime 不需要复杂，但必须清楚表现“这一轮没有白玩”。

## 记忆结构

```ts
type MemoryRuntime = {
  loopCount: number
  knownClues: Record<string, ClueState>
  confirmedFacts: Record<string, FactState>
  eventHistory: RuntimeEvent[]
  loopSummaries: LoopSummary[]
  unlockedRoutes: string[]
}
```

## 每轮结束结算

第一轮失败：

```text
本轮结果：爆炸发生，循环重置。

新增线索：
- 爆炸固定发生在 14:15
- 小宁在爆炸前去过三号车厢
- 小宁听到金属碰撞声

下一轮建议：
直接调查三号车厢。
```

第二轮失败：

```text
本轮结果：爆炸发生，循环重置。

新增线索：
- 三号车厢行李架上有旧帆布包
- 灰衣乘客进入三号车厢后空手返回
- 帆布包可能与爆炸有关

下一轮建议：
拿这些线索说服乘务员陆成。
```

第三轮成功：

```text
本轮结果：三号车厢提前清空，爆炸造成的伤亡被避免。

试玩目标完成：
你已经掌握 LT 的核心循环：
调查 → 记录 → 复盘 → 利用记忆优化路线 → 改变结果。
```

# 十一、AssistantRuntime 设计

许知微的主动性只在前三轮明显出现。

## 主动引导规则

| 场景         | 许知微行为         |
| ---------- | ------------- |
| 第一轮开始      | 主动解释基本目标      |
| 第一轮玩家停滞    | 提醒观察车厢或询问小宁   |
| 第一轮结束      | 总结基础线索        |
| 第二轮开始      | 提醒复用上一轮信息     |
| 第二轮获得帆布包线索 | 解释线索价值        |
| 第三轮开始      | 提醒进入干预阶段      |
| 目标完成       | 简短确认目标价值      |
| 玩家求助       | 根据当前 Goal 给建议 |

## 许知微反馈模板

目标完成时：

```text
“这条线索有价值。它不是孤立信息，它能和上一轮的记录拼起来。”
```

未完成时：

```text
“你已经问到了方向，但还缺关键确认。小宁提到了声音，但我们还不知道声音来自哪里。”
```

路线优化时：

```text
“下一轮可以跳过基础询问，直接去三号车厢。循环的价值就在这里。”
```

# 十三、系统闭环

试玩版的核心闭环应该非常明确：

```text
玩家输入指令
↓
CommandSystem 解析为标准 Action
↓
Runtime 执行动作，产生 Event
↓
MemoryRuntime 写入 Clue / Fact / State
↓
GoalEngine 判定目标是否完成
↓
AssistantRuntime 解释目标价值
↓
UI 更新目标、线索、建议行动
↓
循环失败或成功后，MemoryRuntime 生成复盘
```

这就是 LT 的背后设计逻辑。

试玩版要让玩家自然感受到这套逻辑，而不是靠说明文解释。

# 十四、试玩版最终结尾

成功结尾可以这样写：

```text
14:15。

爆炸声从三号车厢尽头传来。
车窗剧烈震动，灯光瞬间熄灭，又很快亮起。

但这一次，没有尖叫声。

三号车厢已经被清空。
旧帆布包被隔离在连接处，爆炸被限制在狭小空间里。

陆成跌坐在地上，满脸冷汗地看着你。

“你到底是什么人？”

许知微没有立刻回答。

她低头看了一眼自己的笔记本，又看向你。

“寒灯。”

“这不是第一次循环，也不会是最后一次。”
```

然后显示：

```text
试玩版结束。

你已经完成：
✓ 获得线索
✓ 使用记忆
✓ 优化路线
✓ 说服关键 NPC
✓ 改变一次循环结果

正式版中，循环不会只为了阻止爆炸。
你还需要找出：谁制造了事故，谁在利用循环，以及你自己到底是谁。
```

# 十五、我建议你现在就按这个版本开发

0.7 试玩版不要再扩剧情。就做这条最小链路：

```text
第一轮：观察 + 询问 + 失败复盘
第二轮：记忆带入 + 调查三号车厢 + 锁定帆布包
第三轮：线索组合 + 说服乘务员 + 改变结果
```

对应系统能力：

```text
助手 Runtime：许知微主动引导与复盘
记忆 Runtime：保存线索、事件、循环摘要
GoalEngine：结构化目标完成判定
指令系统：自然语言转标准 Action
UI：常驻目标栏 + 建议行动 + 线索反馈
```

这版试玩不复杂，但非常适合展示 LT 的核心：
**玩家不是靠数值成长，而是靠记忆、线索和路线优化，在循环中逐步获得控制力。**
