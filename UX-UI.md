我建议把 LT 的手机竖版 UI 重新定义成一个核心东西：

**玩家不是在“点菜单”，而是在一个 15 分钟循环里调试现场。**

所以 UI 不应该像普通 RPG 那样堆功能按钮，而应该像一个“现场状态面板 + 行动反馈流 + 时间线调试器”。玩家每次点击按钮后，都应该立刻明白三件事：

1. 我刚刚做了什么。
2. 这件事消耗了什么。
3. 世界状态发生了什么变化，以及下一步该看哪里。

---

# 一、当前 UI 的主要问题

从截图看，现在的问题不是“丑”，而是**信息层级混乱**。

## 1. 顶部功能按钮过重

现在「线索 / 人物 / 记忆 / 帮助 / 重置」全部放在顶部黄金区域，会让玩家误以为这些是当前最重要的操作。

但真正核心应该是：

**当前位置 → 当前目标 → 可执行行动 → 行动结果 → 时间线变化**

线索、人物、记忆是辅助信息，不应该和主流程抢视觉权重。

---

## 2. 场景卡太重，行动反馈太弱

当前“二号车厢”场景卡占了很大空间，但玩家点击按钮后的反馈没有形成强烈的“世界状态变化感”。

玩家点了「观察当前位置」之后，理想体验应该是：

> 我执行了一次观察 → 时间推进 / AP 消耗 → 发现了什么 → 新线索入库 → 新行动解锁

而不是只是下面多一段文字。

---

## 3. 按钮类型混在一起

现在按钮里有：

* 人物：小宁、许知微、赵乘警
* 行动：观察当前场景、盯住小宁、守连接处观察
* 移动：前往连接处

这些本质不同，却放在一起，玩家需要自己理解每个按钮意味着什么。

应该拆成：

* **主行动**：当前最推荐、最有价值的行动
* **观察行动**：观察、盯住、等待
* **对话对象**：和谁说话
* **移动行动**：去哪里
* **高风险行动**：提醒乘警、强行检查、拆卸等

---

# 二、推荐的整体布局

手机竖版建议采用这个结构：

```text
┌──────────────────────┐
│ 顶部状态栏：第1轮 / 14:00 / AP10 │
├──────────────────────┤
│ 当前目标卡：证明二号车厢异常 0/1  │
├──────────────────────┤
│ 当前场景卡：二号车厢              │
│ 简短环境描述 + 关键人物状态       │
│ 当前异常提示 / 已知线索摘要       │
├──────────────────────┤
│ 行动反馈流 / 时间线事件流         │
│ 14:00 许知微提醒你                │
│ 14:01 你观察了当前场景            │
│ 14:03 小宁出现异常动作            │
├──────────────────────┤
│ 底部行动区：推荐行动 / 更多行动   │
├──────────────────────┤
│ 输入区：对话 / 行动自由输入       │
└──────────────────────┘
```

核心变化是：

**把行动按钮从场景卡里移出来，放到底部行动区。**

原因很简单：手机竖屏玩家主要靠拇指操作，按钮应该在底部附近，而不是堆在中上部。

# 三、主界面应该只保留四个核心区

## 1. 顶部状态栏

保持简洁：

```text
第 1 轮 · 14:00        AP 10
```

右侧可以保留声音按钮，但不要太突出。

如果未来时间线系统更重要，可以把顶部改成：

```text
第 1 轮    14:00 ━━━●━━━━ 14:15    AP 10
```

用一个很细的时间进度条告诉玩家：

**你正在接近事故发生时间。**

---

## 2. 当前目标卡

现在目标是：

> 证明二号车厢存在异常，并说服赵乘警检查地板。

建议改成更游戏化的目标卡：

```text
当前目标
证明二号车厢异常

需要完成：
□ 找到可疑证据
□ 让赵乘警愿意检查地板
```

这样比单纯 `0/1` 更清楚。

当玩家获得关键证据后：

```text
当前目标
证明二号车厢异常

✓ 找到可疑证据：地板缝隙异常
□ 让赵乘警愿意检查地板
```

玩家会自然知道下一步该干什么。

---

## 3. 当前场景卡

场景卡不应该写太长。它应该是“当前世界状态”，不是小说正文。

建议结构：

```text
二号车厢

列车中段，灯光昏黄。小宁抱着旧布娃娃坐在靠窗位置。
赵乘警正在过道巡逻，许知微在你对面记录。

现场状态：
小宁：紧张，频繁看向座位下方
赵乘警：警惕，但暂未相信你
车厢地板：未检查
```

这里的重点不是文学描写，而是**状态可读**。

每个状态都应该能被玩家利用。

---

## 4. 行动反馈流

这是最重要的区域。

玩家点击任何按钮后，都应该在这里生成一张“行动结果卡”。

例如点击「观察当前场景」后，不要只是追加一段普通文本，而是生成：

```text
14:01 · 行动结果
你观察了二号车厢。

消耗：1 AP / 1 分钟

你注意到小宁一直避开你的目光，但她的右手始终按着布娃娃。
当赵乘警经过时，她明显缩了一下。

新增线索：
+ 小宁害怕赵乘警
+ 小宁反复看向座位下方

解锁行动：
检查小宁座位下方
追问小宁
```

这张卡让玩家非常清楚：

**我做了行动 → 有成本 → 有发现 → 有新选择。**

这比传统对话流更适合 LT。

---

# 四、按钮交互方案

我建议把按钮分为四类，不要混在一起。

## 1. 推荐行动

底部默认只显示 2 到 3 个最推荐的行动。

例如当前状态下：

```text
推荐行动
[观察当前场景] [询问小宁] [前往连接处]
```

其他行动收进「更多」。

这样玩家不会被 8 个按钮压住。

---

## 2. 更多行动抽屉

点击「更多行动」后，从底部弹出行动面板：

```text
更多行动

观察
[观察当前场景] [盯住小宁] [盯住赵乘警]

对话
[询问小宁] [询问许知微] [提醒赵乘警]

移动
[前往连接处] [查看三号车厢]

等待
[等待 1 分钟] [守在连接处观察]
```

这比现在一堆胶囊按钮自然很多。

---

## 3. 高风险行动需要确认

例如：

* 提醒赵乘警
* 强行检查地板
* 抢走小宁布娃娃
* 公开喊有炸弹

这些行动可能改变 NPC 态度或导致失败，应该弹出轻量确认：

```text
高风险行动

你要直接提醒赵乘警检查地板。
如果证据不足，他可能认为你精神异常，并降低信任。

[取消] [继续]
```

普通行动不需要确认，避免打断节奏。

---

## 4. 持续行动要变成“状态”，不是一次性文本

比如「盯住小宁」不是普通按钮，它应该进入一个持续观察状态。

点击后，顶部或场景卡下方出现：

```text
正在盯住：小宁
持续观察中 · 下一次异常动作可能被捕捉
[停止盯住]
```

然后随着时间推进，系统自动产生观察片段：

```text
14:02 · 盯住小宁

小宁低头抱紧布娃娃。
她没有看你，也没有看赵乘警，而是看向自己座位下方。
```

如果继续等待：

```text
14:04 · 盯住小宁

列车轻微晃动时，小宁本能地用脚挡了一下座位下方。
她似乎不想让别人靠近那里。
```

这会让玩家理解：

**盯住不是“点一下获得文本”，而是占用注意力换取时间线证据。**

这和 LT 的时间线玩法非常契合。

---

# 五、几种关键按钮的具体交互设计

## 1. 点击「观察当前场景」

行为：

1. 按钮变成 `观察中...`
2. 时间/AP 扣除
3. 行动结果卡插入事件流顶部
4. 场景状态更新
5. 如果发现新线索，弹出轻量提示

展示：

```text
14:01 · 观察当前场景

你放慢呼吸，重新扫视二号车厢。
小宁的位置、赵乘警的巡逻路线、车厢尽头的圆钟，都和许知微描述一致。

但有一点不对：
小宁并不是单纯害怕，她像是在保护什么东西。

新增线索：
+ 小宁在保护座位下方的某物

新行动：
[检查小宁座位下方] [追问小宁]
```

---

## 2. 点击「盯住小宁」

行为：

1. 进入持续观察状态
2. 场景卡显示“当前关注对象：小宁”
3. 底部推荐行动变成：

   * 等待 1 分钟
   * 靠近小宁
   * 停止盯住
4. 时间推进时自动记录小宁行为

展示：

```text
当前关注对象：小宁

你开始盯住小宁。
她注意到了你的视线，身体变得更僵硬。
```

继续等待后：

```text
14:03 · 小宁异常动作

赵乘警从过道经过时，小宁突然把布娃娃抱得更紧。
她的脚尖轻轻抵住座位下方，好像在挡住什么。
```

新增线索：

```text
+ 小宁试图遮挡座位下方
```

---

## 3. 点击「询问小宁」

不要直接混入普通事件流。建议进入“对话聚焦模式”。

界面变成：

```text
小宁

她低着头，抱着布娃娃，不敢看你。

你想怎么问？

[你刚才在看什么？]
[你是不是知道地板下面有什么？]
[别害怕，我不是乘警]
[自由输入]
```

对话结束后，回到主界面，并生成摘要卡：

```text
14:04 · 对话摘要

你询问了小宁。
她没有直接回答，但在你提到“座位下面”时明显慌了。

人物状态变化：
小宁信任 +1
小宁恐惧 +1

新增线索：
+ 小宁知道座位下方有东西
```

这样对话不会污染主界面，又能留下结构化结果。

---

## 4. 点击「前往连接处」

移动行动应该有清晰转场：

```text
你离开二号车厢，穿过摇晃的车厢连接处。
```

然后场景卡变为：

```text
车厢连接处

风声从门缝里灌进来。这里可以观察二号车厢和三号车厢之间的人员流动。
```

推荐行动随之变化：

```text
[守在连接处观察] [返回二号车厢] [检查车门]
```

重点是：**移动后，按钮必须完全刷新，玩家要感觉自己真的换了位置。**

---

## 5. 点击「提醒赵乘警」

如果证据不足：

```text
证据不足

你现在可以提醒赵乘警，但他未必会相信你。

当前可用证据：
- 小宁紧张
- 小宁看向座位下方

缺少关键证据：
- 地板异常
- 明确危险来源

[继续提醒] [先收集证据]
```

如果证据足够：

```text
14:08 · 说服赵乘警

你把小宁的异常动作、座位下方的痕迹、地板缝隙的位置连起来说给赵乘警听。

赵乘警沉默了几秒，终于蹲下身检查地板。

目标推进：
✓ 让赵乘警检查地板
```

这会让玩家理解“证据链”的价值。

---

# 六、线索 / 人物 / 记忆 / 帮助 / 重置 的处理

现在顶部五个按钮建议改掉。

## 新结构

顶部只保留一个「档案」入口和一个「许知微」入口。

```text
[档案] [许知微]
```

点击「档案」后进入抽屉：

```text
档案

[线索] [人物] [时间线] [循环记忆]
```

点击「许知微」后，不是普通帮助页，而是当前局势建议：

```text
许知微建议

现在不要急着找炸弹。
你还没有证明二号车厢异常。

建议顺序：
1. 观察小宁
2. 检查她座位下方
3. 拿证据说服赵乘警
```

这让“帮助”变成游戏内角色能力，而不是系统菜单。

「重置」放到右上角更多菜单里，不应该和核心玩法并列。

# 七、时间线管理的 UI 表达

既然你已经实现了时间线管理，就要把它做成 LT 的特色，而不是隐藏系统。

建议加入一个“迷你时间线”。

顶部或目标卡下方：

```text
14:00 ━●━━━━━━ 14:15
       当前
```

随着事件发生：

```text
14:00  醒来
14:03  小宁异常
14:06  赵乘警经过
14:10  连接处响动
14:15  事故
```

玩家点击时间线，可以打开：

```text
本轮时间线

14:00 醒来，许知微说明循环
14:01 观察二号车厢，发现小宁紧张
14:03 盯住小宁，发现她遮挡座位下方
14:05 询问小宁，她回避地板问题
```

循环后，应该可以对比：

```text
第 1 轮
14:03 小宁遮挡座位下方

第 2 轮
14:03 玩家提前询问小宁，小宁没有遮挡动作

差异：
小宁行为被玩家干预
```

这就是 LT 最有价值的地方。

# 八、视觉风格建议

整体可以继续保持黑暗、列车、悬疑、金色高亮，但需要更克制。

## 颜色分工

建议固定语义：

```text
金色：主线目标 / 关键推进
蓝色：观察 / 信息
绿色：已获得线索 / 安全行动
红色：危险 / 高风险 / 时间逼近
灰色：不可用 / 已完成
```

不要所有按钮都用金边，否则玩家不知道哪个重要。

## 文本层级

建议：

```text
场景标题：22px 左右
正文：16px-17px
辅助说明：13px-14px
按钮：15px-16px
```

当前截图的正文略大、卡片信息过密。手机上可读性不错，但信息密度偏高。

## 按钮数量

默认状态最多显示 3 个按钮。

超过 3 个，全部进入「更多行动」。

这是关键原则。

# 十、最重要的交互原则

我建议 LT 后续所有 UI 都遵守这几条规则。

## 1. 普通行动不弹窗，结果进入事件流

观察、等待、移动、普通询问，都不要弹窗。

弹窗会打断沉浸。

结果应该进入事件流，并更新状态。

---

## 2. 只有三种情况弹出面板

可以弹出面板的情况只有：

1. 查看档案：线索、人物、记忆、时间线
2. 对话聚焦：和 NPC 深入交流
3. 高风险确认：可能导致失败或重大状态变化

其他时候不要弹。

---

## 3. 每次行动都要显示状态变化

行动结果卡必须包含：

```text
时间变化
AP变化
发现内容
新增线索
人物状态变化
解锁行动
目标推进
```

不一定每次都有全部字段，但系统结构要支持。

---

## 4. “盯住”必须做成持续状态

这是 LT 很重要的玩法。

不要把「盯住小宁」做成普通观察按钮。

它应该像 debugger 里的 watch expression：

```text
Watch: 小宁
```

玩家持续关注她，系统在时间线推进时自动捕捉她的异常行为。

这会非常符合你之前说的“工程感”和“时间线调试感”。

# 十一、建议的组件拆分

从开发角度，可以拆成这些组件：

```text
GameShell
  ├─ StatusBar
  ├─ TimelineMiniBar
  ├─ ObjectiveCard
  ├─ SceneStateCard
  ├─ EventFeed
  │    ├─ DialogueEventCard
  │    ├─ ActionResultCard
  │    ├─ ClueUnlockedCard
  │    └─ TimelineChangeCard
  ├─ ActionDock
  │    ├─ RecommendedActions
  │    └─ MoreActionsSheet
  ├─ FocusWatchBar
  ├─ DialogueFocusSheet
  ├─ ArchiveSheet
  └─ CommandInput
```

行动结果统一用一个结构：

```ts
type ActionResult = {
  actionId: string
  title: string
  time: string
  cost: {
    ap?: number
    minutes?: number
  }
  narrative: string
  cluesAdded?: Clue[]
  npcStateChanges?: NpcStateChange[]
  objectiveChanges?: ObjectiveChange[]
  unlockedActions?: Action[]
  timelineEvents?: TimelineEvent[]
}
```

这样 UI 才能稳定渲染，不会每个按钮都变成特殊逻辑。

# 十二、落地优先级

建议不要一次性重做全部，按这个顺序来。

## 第一阶段：重排主界面

先做：

1. 顶部状态栏简化
2. 当前目标卡结构化
3. 场景卡缩短
4. 行动按钮移到底部
5. 默认只显示 3 个推荐行动

这一步完成后，界面会立刻清爽很多。

## 第二阶段：统一行动结果卡

所有按钮点击后，不再随意追加文本，而是统一生成：

```text
行动结果卡
线索卡
人物状态变化卡
目标推进卡
```

这是体验提升最大的部分。

---

## 第三阶段：实现“盯住 / 守候 / 等待”的持续行动模式

这是 LT 区别于普通文字冒险的关键。

重点做：

```text
盯住小宁
守连接处观察
等待赵乘警经过
观察三号车厢动静
```

这些都应该成为持续状态，而不是一次性按钮。

---

## 第四阶段：档案系统抽屉化

把：

```text
线索 / 人物 / 记忆 / 帮助 / 重置
```

从顶部拿掉。

改成：

```text
档案
许知微建议
更多菜单
```

这样主界面就会真正聚焦到核心玩法。

# 最终建议

LT 的 UI 不应该追求“功能都露出来”，而应该追求：

**玩家每一眼都知道当前局势，每一次点击都看到世界状态变化，每一轮循环都能对比自己改变了什么。**

具体到你现在这个版本，我建议核心改动是：

1. **行动按钮下移到底部行动区。**
2. **默认只展示 2-3 个推荐行动。**
3. **点击行动后生成结构化行动结果卡。**
4. **盯住/守候类按钮改成持续观察状态。**
5. **线索/人物/记忆统一收进档案抽屉。**
6. **许知微变成局势建议入口，而不是普通帮助按钮。**
7. **时间线变成可视化核心，而不是隐藏数据结构。**

这样改完后，玩家会更自然地理解 LT 的玩法：
不是读故事，也不是乱点按钮，而是在一个循环里不断观察、验证、干预、复盘。
