## v0.8.1 — 存档版本检测与 Breaking Change 强制重置

日期：2026-06-20

### 新增

- **存档版本化**：新增 `SaveMeta` 结构（6 字段：`appId`, `saveSchemaVersion`, `runtimeVersion`, `storyVersion`, `createdAt`, `updatedAt`），写入 `lt:save:meta`
- **`lt:` key 体系**：`lt:save:meta`、`lt:save:runtime`、`lt:save:memory`、`lt:save:goals`、`lt:settings`，统一命名前缀
- **版本检测与强制重置**：启动时自动检测旧 `looptrain.standalone.v1` 存档和 `saveSchemaVersion`/`storyVersion` 不匹配，弹出重置引导模态框
- **重置模态框**：区分 legacy 数据检测（「新版试玩已重构」）和 incompatible version（「剧情版本不匹配」）两种场景文案
- **手动重置三路径**：
  - UI 按钮：命令栏「🔄重置」→ 确认对话框 → 清除并重建
  - URL 参数：`?reset=1`，处理完成自动移除参数
  - 开发者控制台：`window.LT_RESET()`，清除后刷新页面
- **旧数据归档**：旧 `looptrain.standalone.v1` 数据归档到 `lt:legacy:<timestamp>`，开发侧保留可调试
- **IndexedDB 旧库清理**：启动时异步删除 `LoopTrainDB`、`LoopTrainRuntimeDB`、`LoopTrainMemoryDB`，新版使用 `LoopTrainMemoryDB_v0_8` 隔离
- **版本升级分级策略**：Patch（不重置）→ Minor（可写 migration）→ Major/Breaking（必须重置）

### 变更

- 存档从单 key `looptrain.standalone.v1` 迁移到双 key `lt:save:meta` + `lt:save:runtime`
- `init()` 流程重构为 4 路径分流：新玩家、旧数据归档、版本不兼容强制重置、兼容存档正常恢复
- `AudioManager` 静音设置迁移到 `lt:settings`（`{ muted: boolean }`），跨版本保留
- `resetGame()` 升级为新存档系统（`detectLegacyKeys → archiveLegacyData → clearLtKeys → initNewSave`）
- 数据安全：清除操作只匹配 `lt:` 前缀且排除 `lt:legacy:` 和 `lt:settings`，不清空整个 `localStorage`

### 影响

- **BREAKING**：旧存档（`looptrain.standalone.v1`）完全不兼容，首次进入新版时强制重置
- 旧数据归档保留在 `lt:legacy:<timestamp>`，玩家不可见，开发侧可调试
- 用户设置（静音偏好）跨版本保留
- 本版不做旧数据迁移，后续 Minor 版本可写 migration 逻辑
