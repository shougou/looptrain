## Added
- 时间线回放系统：失败后从上一轮时间线选择锚点接入
- NPC 记忆残响系统：跨轮情感残留影响开场白和 LLM 表演
- LLM Echo Guard：禁词检测 + 回退 Mock 模式
- Runtime 事件记录系统：IndexedDB 前端封装（5 store + 降级策略）
- 四层版本号体系：app / channel / release / story
- 3 个新引擎测试（replay_resume / npc_memory_echo / llm_echo_guard）
- POST /api/replay/resume 端点
- 前端 ReplayAnchorPicker UI（失败结算卡内嵌）

## Changed
- engine.js startDialogue() 使用 resolveNpcOpening() 替代 npc.opening
- server.js /api/health 返回四层版本字段
- app.js 版本常量迁移到四层体系
- 版本号从 v0.11.0 升级到 v0.12.0-replay-echo
- Save Schema Version 从 1 升级到 2（breaking change）

## Fixed
- portrait-intro.js getBoundingClientRect is not a function（47 次控制台错误）
- nextLoop() 遗漏 saveState() 导致刷新后丢失新轮次状态
