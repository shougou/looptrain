## 背景

LoopTrain v0.4.3 试玩版已经建立了基本的游戏循环：玩家进入列车场景 → 与 NPC 对话 → 探索 → 发现线索 → 循环重启。但这个版本仍然存在若干影响体验的问题。

本文整理当前已知的所有问题，用于后续版本规划和优先级判断。

## 当前问题

### 1. SillyTavern 原始界面仍可能显示

在不带 `?looptrain=game` 参数访问时，用户看到的是 SillyTavern 的标准管理界面，而非 LoopTrain 的游戏覆盖层。即使是玩家模式，首次加载时 ST 原始界面可能短暂闪现。

**影响**：试玩用户感到困惑，破坏游戏沉浸感。

**当前状态**：v0.4.3 已经支持 Game Shell 模式（`?looptrain=game`），但首次加载的界面闪现问题尚未彻底解决。

### 2. 小宁立绘资源 404

小宁的立绘资源在部分部署环境下返回 404。原因是立绘路径在早期的 Extension 版本中指向了未部署的旧路径。

**影响**：最核心的对话角色没有立绘显示。

**当前状态**：v0.4.3 已将立绘资源路径统一到 `/scripts/extensions/third-party/LoopTrain/`，但需要验证所有部署环境。

### 3. 移动端遮罩和开场字幕问题

移动端开场字幕在不同屏幕尺寸下存在错位、裁剪和字体过小的问题。

**影响**：移动端是新用户的第一个视觉印象，开场体验不佳直接影响留存。

**当前状态**：需要针对 iPhone 14 (390×844) 和 iPhone 16 Pro Max (430×932) 等主流手机竖屏尺寸进行专项适配。

### 4. 音效系统完全缺失

当前游戏完全没有声音。按钮点击、对话进行、倒计时警告、列车爆炸、循环重启等关键节点都没有任何音效反馈。

**影响**：沉浸感和时间压力感知大幅削弱，列车环境氛围无法建立。

**当前状态**：计划在下一阶段建立最小可用音效系统。详见《准备增加音效系统》。

### 5. Server Plugin 连接问题

Server Plugin 未连接时，游戏状态无法持久化，部分结算逻辑回退到本地 Mock。

**影响**：复杂剧情状态管理不稳定，循环之间的状态继承可能丢失。

**当前状态**：需要确认 `config.yaml` 中 `enableServerPlugins: true` 是否正确配置，以及 `plugins/looptrain` 是否正确部署。

### 6. NPC 剧情链路不完整

当前试玩版中三个主要 NPC（小宁、赵乘警、沈墨寒）的对话和行动链路尚未全部完成。部分线索触发条件、NPC 自主时间线行为仍为占位。

**影响**：玩家在试玩过程中可能遇到"不知道该做什么"的尴尬空白期。

**当前状态**：正在进行中，目标是完成 3 个 NPC + 1 个隐藏 NPC 的完整试玩链路。

## 本次调整

本次主要是问题整理和优先级判断，不做代码修改。

## 设计判断

问题的优先级排序依据：

1. **立绘 404 和界面暴露 → 最高优先级**：直接阻碍新用户体验游戏。
2. **移动端遮罩 → 高优先级**：移动端是核心目标平台。
3. **音效系统 → 中优先级**：影响沉浸感但不阻碍核心玩法。
4. **NPC 链路完整性 → 中优先级**：影响游戏深度而非基础可用性。
5. **Server Plugin 连接 → 中优先级**：有本地 Mock 回退方案。

## 后续计划

1. 首先修复立绘资源路径和界面闪现问题。
2. 专项适配移动端开场字幕和遮罩。
3. 建立最小可用音效系统。
4. 完成 NPC 试玩链路。
5. 稳定 Server Plugin 连接。
