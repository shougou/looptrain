# LoopTrain-ST v0.3.1 Development Notes

## 本版目标

v0.3.1 在 v0.3 的基础上补充两个试玩版关键 UX：

1. 打开游戏第一眼显示背景介绍。
2. 在输入区加入「扮演 / 指令」页签。

继续保持原则：不修改 SillyTavern 核心代码，只通过外层 UI Extension、Server Plugin 和 `chatMetadata.looptrain` 管理游戏状态。

## 开场背景页

首次打开 LoopTrain 覆盖层时显示电影字幕式开场：

- 1947 年冬
- 重庆开往江城的渝江线 307 次夜行列车
- 玩家公开身份：普通乘客
- 玩家隐藏身份：打入敌人内部的地下工作者
- 当前任务：携带绝密情报前往江城，与代号“扣子”的同志接头
- 危机：09:00 前列车将在北江铁桥前爆炸
- 第一记忆：第七节车厢地板下方的滴答声

状态字段：

```json
{
  "flags": {
    "intro_seen": false
  }
}
```

点击「开始第 1 轮」后：

- `flags.intro_seen = true`
- 写入 `chatMetadata.looptrain`
- 在主日志中插入开场摘要

## 输入通道：扮演 / 指令

v0.3.1 引入与 ST 输入习惯对应的双通道设计：

| 通道 | 探索模式 | 对话模式 |
|---|---|---|
| 扮演 | 描述行动 | 对当前 NPC 说话 |
| 指令 | 查看线索、查看人物、查看状态、失败测试、重置本轮 | 结束对话、查看线索、查看人物、查看状态 |

状态字段：

```json
{
  "input_channel": "roleplay"
}
```

### 扮演通道

扮演通道进入原有玩法：

- 探索模式：走 `/action/commit`
- 对话模式：走 `/dialogue/message`

### 指令通道

指令通道由 UI Extension 直接拦截，不进入 NPC 对话：

- 查看线索
- 查看人物
- 查看状态
- 结束对话
- 失败测试
- 进入下一轮
- 重置本轮

也支持 `/lt` 风格命令，例如：

```text
/lt status
/lt clues
/lt next
```

## 验收路径

1. 打开 `mock-harness/index.html`。
2. 首屏应显示「第七节车厢」背景开场页。
3. 点击「开始第 1 轮」。
4. 底部可切换「扮演 / 指令」。
5. 扮演模式输入「和小宁对话」进入对话。
6. 指令模式输入「查看线索」不进入 NPC，对话中输入「结束对话」会触发对话结算。
7. 失败后可在指令模式点击「进入下一轮」。
