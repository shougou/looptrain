# LoopTrain-ST UX/UI 设计文档

## 1. UX/UI 总目标

LoopTrain-ST 的 UI 不是普通聊天界面，而是一个手机端优先的 AI 互动故事游戏界面。

目标体验：

```text
打开就是游戏。
输入就是行动。
对话有角色。
失败有记忆。
结算有方向。
```

## 2. 首屏体验

首次打开 LoopTrain 覆盖层时，显示电影字幕式开场页。

目的：

- 告诉玩家时代。
- 告诉玩家地点。
- 告诉玩家身份。
- 告诉玩家危机。
- 告诉玩家目标。

首屏信息：

```text
1947 年冬
重庆 → 江城
渝江线 307 次夜行列车
玩家是地下工作者
09:00 前列车会爆炸
第七节车厢地板下方有滴答声
```

按钮：

```text
开始第 1 轮
```

## 3. 整体布局

手机端竖屏布局：

```text
┌─────────────────────┐
│ 08:45 | AP 10 | 第1轮 │ 顶部状态栏
├─────────────────────┤
│ 场景卡               │
│ 当前目标             │
│ 可见 NPC             │
├─────────────────────┤
│ 日志 / 对话内容       │
│ NPC 前景立绘          │
├─────────────────────┤
│ 扮演 | 指令           │
│ 建议句               │
│ 输入框 | 麦克风 | 发送 │
└─────────────────────┘
```

## 4. 顶部状态栏

展示：

- 当前时间。
- AP。
- 当前轮次。
- 当前模式。
- 当前输入通道。

示例：

```text
08:45｜AP 10｜第 1 轮    探索｜扮演
```

对话模式示例：

```text
08:45｜AP 7｜第 1 轮     对话：小宁｜扮演
```

## 5. 场景卡

展示：

- 当前地点。
- 当前背景。
- 当前目标。
- 可交互 NPC。

当前目标根据证据数量动态变化：

| 条件 | 目标 |
|---|---|
| 0 条证据 | 证明第七节车厢存在异常，并说服赵乘警检查地板 |
| 1 条证据 | 已获得 1 条有效证据，还需要更多证据 |
| 2 条证据 | 证据已足够，可以说服赵乘警 |
| 成功 | 试玩版已完成 |

## 6. 探索 / 对话模式

### 探索模式

玩家输入的是行动。

示例：

```text
我假装系鞋带，低头检查座位下方。
我走到小宁身边，蹲下来和她说话。
我走向沈墨寒，试探他是否知道连接处发生过什么。
```

### 对话模式

玩家输入的是台词或带动作的台词。

示例：

```text
别怕，我也听见了那个声音。你刚才是不是看见了什么？
赵乘警，我不是要制造恐慌，但我已经确认声音不在座位下面。
沈墨寒，你刚才为什么在连接处停了一下？
```

## 7. 扮演 / 指令页签

底部输入区上方显示：

```text
扮演（对话）｜指令（行动）
```

### 扮演通道

扮演输入进入故事。

- 探索模式：描述行动。
- 对话模式：对 NPC 说话。

### 指令通道

指令输入不会被 NPC 听见。

支持：

```text
查看线索
查看人物
查看状态
结束对话
失败测试
进入下一轮
重置本轮
```

也支持：

```text
/lt status
/lt clues
/lt next
```

第一次切换提示：

```text
指令不会被 NPC 听见，可用于查看线索、结束对话或进入下一轮。
```

## 8. NPC 立绘

对话模式下显示 NPC 前景立绘。

要求：

- 电影感。
- 半透明前景遮罩。
- 不遮挡输入区。
- 对话内容可读。
- 有轻微浮动。
- 小宁妈妈作为隐藏记忆节点显示柔化立绘。

NPC：

| NPC | 立绘 |
|---|---|
| 小宁 | 小女孩抱旧布娃娃 |
| 赵乘警 | 铁路乘警 |
| 沈墨寒 | 灰大衣男人 |
| 小宁妈妈 | 隐藏记忆形象 |

## 9. 建议句

建议句不是直接执行，而是填入输入框。

优点：

- 玩家可以修改。
- 保留自由输入。
- 降低新手门槛。

探索模式建议：

```text
检查座位下方
和小宁对话
找赵乘警
试探沈墨寒
```

对话模式建议：

```text
温和询问
提到滴答声
问布娃娃
结束对话
```

指令模式建议：

```text
查看线索
查看人物
查看状态
结束对话
失败测试
进入下一轮
重置本轮
```

## 10. 对话结算卡

对话结束后显示：

- NPC 名称。
- AP 消耗。
- 时间推进。
- 对话轮数。
- 获得线索。
- 世界推进。
- 下一步行动。

目的：

```text
把自然语言对话转成可行动的游戏状态。
```

## 11. 失败结算卡

失败后显示 NG 画面与失败结算。

内容：

- 失败原因。
- 带入下一轮的记忆。
- 疑点。
- 下一轮建议。
- 进入下一轮按钮。

目的：

```text
失败不是终点，而是下一轮推理资源。
```

## 12. 成功结算页

成功后显示完整试玩版结束页。

内容：

- 你证明了什么。
- 你还不知道什么。
- 正式版目标是什么。

目的：

```text
让试玩版成为完整体验，而不是测试流程突然结束。
```

## 13. 手机端注意事项

后续真实设备验证重点：

- 输入框是否被键盘遮挡。
- 建议句是否容易点击。
- 立绘是否遮挡文字。
- 失败页是否可滚动。
- 开场页文字是否过长。
- 扮演 / 指令页签是否易理解。
