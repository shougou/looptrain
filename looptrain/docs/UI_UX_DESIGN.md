# LoopTrain Standalone UI/UX 设计说明

## 1. 总目标

LoopTrain Standalone 的 UI 是一个手机端优先的互动叙事游戏界面。

目标体验：

```text
打开就是游戏。
输入就是行动。
对话有角色。
失败有记忆。
结算有方向。
```

## 2. 布局结构

手机端竖屏布局：

```text
┌─────────────────────┐
│ 08:45 | AP 10 | 第1轮 │ 顶部状态栏
├─────────────────────┤
│ 场景卡               │
│ 当前目标             │
│ 可见 NPC             │
├─────────────────────┤
│ 日志 / 对话内容       │
│ NPC 前景立绘          │
├─────────────────────┤
│ 扮演 | 指令           │
│ 建议句               │
│ 输入框 | 发送         │
└─────────────────────┘
```

## 3. 对话布局

点击顶部 NPC 后：

- 立绘显示在中间偏下。
- 对话卡片显示在底部上方。
- 输入栏固定在屏幕底部。
- 不得把对话框贴到屏幕顶部。
- 不得让立绘被整块面板遮住。

## 4. 失败布局

失败时显示全屏结算层：

- 失败原因
- 本轮确认信息
- 带入下一轮的记忆
- 下一轮建议
- 进入下一轮按钮

## 5. 视觉基调

- 深色背景
- 列车、隧道、夜间、低照度
- 克制、冷静、悬疑
- 不使用商业官网风格
- 不使用过度赛博朋克效果

## 6. 可访问性

- 按钮必须可点击且可聚焦。
- 文本对比度足够。
- 手机端输入区不能遮挡主要操作。
- 动效应克制。

## 7. 本地验证

使用：

```bash
bash scripts/verify_slt.sh
```

并手动检查：

```text
进入第七节车厢
→ 点击顶部 NPC「小宁」
→ 立绘和对话卡片布局正确
```
